My very own little website

Why?

Why do I need a site? Because it’s 2014. Because I think I’ve got something to say. Because I enjoy moving pixels around on a screen. Because I didn’t want to design my resume in Word or bother learning InDesign, so I made it in HTML and decided I needed a site to host it on.

GitHub Pages

I got frustrated with the wonky workflow I had to use to manage my code on GitHub, host it elsewhere, and design the site in WordPress’s slow and clunky environment. Luckily I found GitHub Pages and now my life is wonderful.

repo: ryantuck.github.io

Jekyll

Jekyll is awesome. I infinitely prefer it to WordPress. It’s super easy to use and I get to open up the command line and feel like I’m Neo. I write most to all of my pages in Markdown, and can add bits here and there in html using Liquid tags.

Font Awesome

I use font awesome a lot. It’s nifty.

Simple Grid

I came across Simple Grid looking for just that - a simple grid. And it’s nice, simple, and responsive. I use it on my resume.

Trello

That Trello link in the footer will take you to my board for managing tasks related to the site. I’ve found Trello to be the best of all possible task managers out there. But I’ve got ideas for better ones.
