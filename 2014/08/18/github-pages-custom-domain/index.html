Setting up GitHub pages with a custom domain

Aug 18, 2014

I frequently spend time bullshitting around with details instead of focusing on more important matters. Like organizing a heap of hundreds of nails, screws, nuts, and bolts into neat little drawers instead of making myself dinner, going to the gym, or literally anything else more important.

So, this morning, I decided I needed to set up my custom domain forwarding for my github pages site. My site repo is hosted at ryantuck.github.io, and I own the domain ryantuck.io. I wanted to set it up so typing in any of these three domains into my browser:

ryantuck.io

www.ryantuck.io

ryantuck.github.io

would all redirect to ryantuck.io.

After much dicking around, I figured it out. Here’s what you need to do:

  1. add a CNAME file to the repo
  2. change the A record and CNAME record on the DNS end
  3. set up a 301 redirect

I use Bluehost, and this was no problem. Here’s the gist of it:

CNAME:

ryantuck.io

DNS stuff:

A records:
@ - 192.30.252.153
@ - 192.30.252.154

CNAME:
www - ryantuck.io

301 Redirect:
ryantuck.io redirects to ryantuck.github.io
